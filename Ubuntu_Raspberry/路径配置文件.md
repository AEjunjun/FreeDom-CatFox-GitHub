## ⚙️ 自启动服务配置 (systemd)

所有服务文件位于 `/etc/systemd/system/`

| 服务名称 | 配置文件 | 说明 |
|---------|---------|------|
| `sensor_collector.service` | `/etc/systemd/system/sensor_collector.service` | 传感器采集自启动服务 |
| `timed_photo.service` | `/etc/systemd/system/timed_photo.service` | 定时拍照服务（每5分钟拍一次） |
| `mqtt_capture.service` | `/etc/systemd/system/mqtt_capture.service` | MQTT指令监听服务 |

### 常用服务管理命令
```bash
# 查看服务状态
sudo systemctl status sensor_collector
sudo systemctl status timed_photo
sudo systemctl status mqtt_capture

# 重启服务
sudo systemctl restart sensor_collector

# 查看日志
sudo journalctl -u sensor_collector -f
```

## 📡 MQTT 通信

- **Broker 地址**: `39.106.159.23:1883`
- **订阅主题**: `farm/capture` （树莓派订阅，接收拍照指令）
- **发布主题**: `farm/capture/result` （树莓派发布拍照结果）

### 拍照流程
```
用户点击拍照 → Flask API → MQTT publish "farm/capture" 
→ 树莓派 mqtt_capture.py 收到 → 拍照 + 采集传感器 → 上传照片 
→ MQTT publish "farm/capture/result" → Flask 更新页面
```

## 🔧 快速修改指南

### 修改传感器采集间隔
编辑 `/home/pi/sensor_collector.py` 中的 `INTERVAL` 变量（单位：秒）
```python
INTERVAL = 900  # 15分钟 = 900秒
```

### 修改拍照间隔
编辑 `/etc/systemd/system/timed_photo.service` 中的 `OnBootSec` 和 `OnUnitActiveSec`

### 修改照片保存数量
编辑 `/root/photo_cleanup.sh` 中的 `MAX_PHOTOS` 变量
